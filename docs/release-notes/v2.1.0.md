# v2.1.0 — 사용량 알림 + Detail 모드 성능

위젯이 *passive 표시*에서 *active 알림*으로. 헤비 사용자의 Detail 모드 latency 도 mtime 캐싱으로 단축.

## 새 기능

### 사용량 임계치 OS 알림 (5h 세션 + 7일 주간, 85% / 95%)
**5시간 세션** 과 **7일 주간 한도** (전체 모델 합산) 사용량이 각각 **85% / 95%** 에 도달하면 OS native notification 발송. 위젯이 Mini 모드거나 다른 작업 중이어도 받을 수 있음. 같은 블록 안에서 *각 임계치당 1회*만 발송 — 매 sync 마다 반복 X.

- **첫 임계치 도달 시점에 권한 요청** — 부팅 시 미리 묻지 않음. 사용자가 실제로 임계치 근접해야 알림 prompt.
- **권한 거부 시** — 세션 동안 재요청 안 함 (OS 캐시 + IPC 낭비 회피). OS 설정에서 변경 후 위젯 재시작 시 자동 인식.
- **알림 enable/disable 은 OS 권한이 유일한 제어** — 위젯 자체 토글 없음 (Settings UI 단순화, OS 알림 설정과 중복 회피).
- 세션·주간 블록 reset 시점에 자동으로 다시 발송 가능 상태.

### 로그 회전 정책 변경
이전 `KeepAll` → `KeepSome(5)`. `widget.log` 누적 디스크 사용 ~5MB 상한 (이전엔 1년 사용 시 수십~수백 MB 누적 위험). 최근 sync · error · update 기록은 버그 리포트용으로 충분히 보관.

### UX 폴리시
- **OS 다크 모드 자동 따라가기** — 사용자가 Settings 에서 명시적으로 토글하기 전까지 OS `prefers-color-scheme` 추종. OS 테마 바꾸면 위젯도 즉시 반영.
- **Donut 클릭 = 지금 동기화** 일관성 — Normal 외 Mini · Detail (Active 카드) 의 hero donut 도 클릭 시 sync. 3 모드 모두 같은 동작.
- **커스텀 스크롤바** — 기본 WebView 스크롤바 (gray slab) 가 Liquid Glass 패널과 톤 안 맞음. 8px 폭 + 투명 트랙 + fill-token thumb (hover 시 강조). Settings 패널 · Detail view 등 overflow 영역 모두 적용.

## 변경

### Detail 모드 풀스캔 → mtime 기반 캐싱
이전엔 Detail 모드 진입 + 매 sync 마다 `~/.claude/projects` 전체 jsonl 재파싱. 헤비 사용자 (수 GB 누적) 환경에서 첫 진입이 *수 초*. v2.1.0 부터는 *파일 mtime 비교*로 변경된 파일만 재파싱, 위젯 lifetime 동안 in-memory 캐시 유지.

- 첫 호출은 기존과 동일 (cold cache).
- 이후 호출은 *변경된 파일만* 다시 읽음 — `widget.log` 의 `aggregate: scanned N files (cache hits=H misses=M) in Tms` 라인으로 측정 가능.
- 삭제된 파일은 캐시에서 자동 제거 (디스크 ↔ 캐시 일관성).

### 버전 헬퍼 스크립트
6 파일 (`package.json` / `package-lock.json` / `tauri.conf.json` / `Cargo.toml` / `Cargo.lock` / `store.ts`) 의 버전 한 줄로 일괄 bump.

```bash
node scripts/bump-version.mjs 2.1.0      # bump
node scripts/bump-version.mjs --check    # 현재 동기화 상태만 확인
```

회귀 사례 §3 (5곳 동시 bump 누락 → 인스톨러 파일명·매니페스트 어긋남) 의 재발 방어.
