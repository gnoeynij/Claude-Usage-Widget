# v2.0.3 — 설정 persist + 트레이 i18n + 에러 배너 4종

**v2.0.0 부터 누적된 *조용한 데이터 손실 회귀* 를 해소.** 사용자가 변경한 lang / dark / opacity / sync interval / always-on-top / mode 가 위젯 재시작 시 *전부 default 로 복원* 되던 문제. 추가로 트레이 메뉴 한국어 표시 + 에러 배너 4종 확장.

## 수정

### 설정 persist gap ([회귀 사례 §15](../../CLAUDE.md))

v2.0.0~v2.0.2 동안 [`store.ts`](../../src/state/store.ts) 의 `setLang` / `setDark` / `setAlwaysOnTop` / `setSyncIntervalMin` / `setOpacity` / `setMode` 6 setter 가 모두 메모리 (`setStore`) 만 갱신하고 `tauri-plugin-store` 에 *persist 하지 않음*. 위젯 재시작마다 lang `en` · dark `false` · opacity `0` · syncIntervalMin `5` · alwaysOnTop `false` · mode `normal` 로 복원.

- `persistSetting` / `loadSetting` generic helper 도입.
- 6 setter 모두 `void persistSetting(key, value)` 호출 추가.
- `initStore` boot 흐름에 6 키 load. `suppressPersist` 플래그로 read-after-write loop 차단.
- `mode` load 는 `setStore` 만 호출해 `applyModeSize` 두 번 실행 race 회피.

### PyQt6 → tauri-plugin-store 마이그레이션 정렬

[`migration.rs`](../../src-tauri/src/migration.rs) 가 v1.5 PyQt6 QSettings 키를 `app.store("settings.json")` 에 마이그레이션했지만, store.ts 는 `widget-settings.json` 에서 읽어 *마이그레이션이 사실상 dead code*.

- 파일명 `settings.json` → `widget-settings.json` 통일.
- PyQt6 snake_case 키 → store.ts camelCase 키 매핑 (`always_on_top` → `alwaysOnTop`, `dark_mode` → `dark`, `bg_opacity` → `opacity`, `mini_mode` + `detail_mode` → `mode` 문자열).
- v1.5.1 `sync_interval` (seconds, `DEFAULT_SYNC_INTERVAL_SEC = 600`) → `syncIntervalMin` (minutes, `/60` 변환). `SYNC_INTERVAL_OPTIONS_SEC = (0, 300, 600, 1800, 3600)` ↔ v2 SegmentedControl `(0, 5, 10, 30, 60)` 완벽 매칭.
- 기존 marker (`%LOCALAPPDATA%\com.gnoeynij.claude-widget\.migrated_from_pyqt6`) 가 이미 찍힌 사용자는 1회 marker 삭제 후 재실행 필요.

## 새 기능

### 트레이 메뉴 i18n ([`tray.rs`](../../src-tauri/src/tray.rs))

v2.0.0 부터 트레이 메뉴 라벨이 영어 hardcode 였음. 패널은 한국어인데 트레이만 영어인 불일치.

- `TrayLabels` struct + `tray_labels(lang)` 분기.
- 한국어: 표시/숨김 · Mini/Normal/Detail 모드 · 지금 동기화 · 종료.
- [`lib.rs`](../../src-tauri/src/lib.rs) boot 시점에 `widget-settings.json` 의 `lang` 읽어 `tray::setup(app, &lang)` 전달.
- lang 변경 후 메뉴는 *재시작 시* 반영 (Tauri 2 menu re-create 비용 회피).

### 에러 배너 4종 ([`ErrorBanner.tsx`](../../src/components/ErrorBanner.tsx))

v2.0.0 부터 `errorCode` 는 4종 (`TOKEN_EXPIRED` / `NO_CREDENTIALS` / `RATE_LIMITED` / `NETWORK`) 정의·발행 됐지만 UI 는 *`TOKEN_EXPIRED` 하나만 렌더*. 나머지 3종은 silent fail.

- 4종 switch + tone (warn / info / danger) 분기.
- lucide 아이콘 매칭: AlertTriangle (TOKEN_EXPIRED) · AlertCircle (NO_CREDENTIALS) · Clock (RATE_LIMITED) · WifiOff (NETWORK).
- i18n en/ko 6 키 추가 (`noCredentials` · `rateLimited` · `networkError` + `*Hint`).
- `danger` tone (NETWORK) 만 빨강, 나머지는 기존 accent tint 재사용.

## 마이그레이션 노트 (v2.0.0~v2.0.2 → v2.0.3)

- v2.0.x 사용자의 `widget-settings.json` 의 `modeSizes` / `breatheEnabled` 는 *그대로 유지*. 새 6 키는 *첫 변경 시 추가됨* (boot 시점에 default 값으로는 안 쓰임 — 사용자가 lang/dark 등을 *한 번이라도* 토글해야 디스크에 기록).
- v1.5.x → v2.0.3 직행 사용자만 PyQt6 마이그레이션 코드가 실행됨. 본인이 이미 v2.0.x 를 부팅한 적이 있다면 marker 가 찍혀 *마이그레이션 skip*. 의도적으로 PyQt6 데이터를 회수하고 싶다면 `%LOCALAPPDATA%\com.gnoeynij.claude-widget\.migrated_from_pyqt6` 삭제 후 위젯 재실행.
