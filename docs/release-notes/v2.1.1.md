# v2.1.1 — 배경 투명도 시각 정합성 hotfix

v2.1.0 release 직후 사용자 신고로 발견된 배경 투명도 (opacity slider) 의 *부분 paint 누락* 해소. v2.0.0 부터 누적된 회귀로 v2.1.0 의 변화 없음 — 다만 사용자가 *라이트 모드 + opacity 100% 조합*을 처음 만져본 시점에 드러남.

## 수정

### 배경 투명도 슬라이더가 위젯 본체를 완전히 fade 시키지 못하던 회귀
opacity 100% 설정 시 위젯 본체 *메인 배경*은 투명되지만 *외곽선 (specular border) + 위쪽 inner glow + 카드 외곽선*이 그대로 paint 되어 *위젯이 안 사라지는 듯*한 모습. 사용자 시각엔 "위젯은 그대로인데 옵션 패널만 fade" 로 인지.

- `.glass-panel::before` (1px specular border) — `opacity: var(--bg-alpha-mult)` 적용
- `.glass-panel::after` (위쪽 inner glow) — `opacity: calc(0.55 * var(--bg-alpha-mult))` 적용
- `.glass-card::before` (카드 외곽선) — `opacity: var(--bg-alpha-mult)` 적용

→ opacity 100% 에서 모든 paint layer 가 함께 사라지고 Settings 오버레이 scrim 만 남음.

### 라이트 모드에서 Settings scrim 이 어두운 회색으로 보이던 회귀
Settings 오버레이 wash 가 `rgba(0, 0, 0, 0.32)` 고정 검은 톤이라 *라이트 모드 + opacity 100%* 조합에서 위젯 본체가 사라진 뒤 *검은 wash 만 남아 desktop 위 어두운 회색* 으로 보임. 다크 모드에선 자연스러웠고 라이트 모드에서만 문제.

- `--scrim-bg` 토큰 도입 — 라이트 `rgba(255, 255, 255, 0.55)` (밝은 wash) / 다크 `rgba(0, 0, 0, 0.32)` (기존 그대로)
- SettingsPanel 오버레이 inline rgba → `var(--scrim-bg)`

→ 라이트 모드 opacity 100% 시 *흰 wash + backdrop blur* 로 자연스럽게 paint.

## 변경

### CLAUDE.md 회귀 사례 §16 + UI SOP "opacity 영역 6 컷 의무"
opacity slider 가 *6 layer 합성* 영역이라 한 layer fix 가 다른 layer leak 을 가림. 향후 PR 에서 opacity 영역 변경 시 **라이트/다크 × opacity 0%/50%/100% = 6 컷 캡처 자기 보고 의무** 추가. 회귀 사례 §16 에 v2.1.1 의 두 fix 와 재발 방어 단서 기록.
