# v2.4.4 — Projection accuracy, Mini error visibility, chart fix

A maintenance release that tames overly aggressive weekly projections, surfaces
sync errors in Mini mode that were previously silent, and fixes a daily-cost
chart bug where the peak day could render shorter than smaller days.

## Fixed

- **Weekly projection no longer over-extrapolates from short bursts.** A burst
  of activity during work hours was projected forward at that same pace into
  nights and weekends, making the "on pace to exceed" warning fire too early.
  Weekly projection now uses week-to-date average only.
- **Mini now surfaces token-expiry / auth / network errors.** Previously Mini
  mode showed no indication when sync failed — only Normal/Detail did. A badge
  plus an info overlay now distinguish at-risk (yellow triangle) from hard
  errors (red circle), and the warning badge/overlay no longer gets stuck open
  after the underlying warning clears.
- **Daily cost chart bars are now proportional to actual cost.** The $ label
  above each bar was eating into the bar's own height, so a peak day with a
  label could render *shorter* than a smaller, unlabeled day (visible in the
  30-day view). Labels are now positioned independently of bar height.
- **Usage panel no longer goes blank on a null `seven_day_sonnet` response.**
  Since the June 23 model-gating change, the usage endpoint can return `null`
  for that block; the client now parses it correctly instead of failing the
  whole response, and shows a generic warning banner instead of a silent red
  status dot.
- **Auto-sync is no longer silently disabled for intervals under 60 seconds.**
  The seconds→minutes conversion during settings migration truncated instead
  of rounding, turning any sub-minute interval into 0 (off).
- **Token-expiry notice now includes a reauth hint** (`/login` in the CLI).

## Notes

- All costs are *estimates* derived from local JSONL × official pricing
  (subscribers are not billed per-token).

## 수정

- **주간 투영이 짧은 burst로 과도하게 외삽되지 않습니다.** 업무시간의 활동
  burst가 밤·주말까지 같은 속도로 투영돼 "한도 초과 예상" 경고가 너무 일찍
  뜨던 문제를 완화했습니다. 이제 주간 투영은 week-to-date 평균만 사용합니다.
- **Mini 모드가 토큰 만료·인증·네트워크 오류를 표면화합니다.** 이전에는
  동기화 실패 시 Normal/Detail에만 표시되고 Mini는 아무 표시가 없었습니다.
  배지와 info 오버레이로 주의(노란 삼각형)와 오류(빨강 원형)를 구분하고,
  경고가 해소되면 오버레이가 열린 채로 갇히지 않습니다.
- **일별 비용 차트 막대가 실제 비용에 정확히 비례합니다.** 막대 위 $라벨이
  막대 자체 높이를 잠식해, 라벨이 붙은 peak일이 라벨 없는 더 작은 날보다
  오히려 짧게 그려지던 문제(30일 보기에서 보임)를 고쳤습니다. 라벨은 이제
  막대 높이와 독립적으로 배치됩니다.
- **`seven_day_sonnet` null 응답에 usage 패널이 빈 화면이 되던 문제 수정.**
  6/23 모델 게이팅 변경 이후 해당 API가 이 블록에 null을 반환할 수 있는데,
  이제 정상 파싱하고 조용한 빨강 상태 점 대신 일반 경고 배너를 표시합니다.
- **60초 미만 동기화 주기가 조용히 꺼지던 문제 수정.** 설정 마이그레이션의
  초→분 변환이 반올림 대신 절삭돼, 1분 미만 주기가 전부 0(꺼짐)이 됐습니다.
- **토큰 만료 안내에 재로그인(`/login`) 안내를 추가했습니다.**
