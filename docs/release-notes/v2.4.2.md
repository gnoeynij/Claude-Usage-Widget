# v2.4.2 — Projection polish, always-on-top fix & faster aggregation

A maintenance release: the limit-projection warnings get smarter and calmer,
always-on-top finally sticks after a restart, and cost aggregation does less
work for heavy users.

## Fixed

- **Always on top** now applies correctly after a restart — previously the pin
  survived the toggle but silently failed to take effect on boot (a re-assert
  that the window layer folded into a no-op).

## Changed — limit projection

- **Relaxes on its own.** A burst's aggressive "on pace to hit the limit"
  estimate now eases off over time between syncs — no need to keep refreshing.
- **Weekly bursts are capped** at 2× your week-to-date average, so a brief spike
  no longer reads as "limit in 7h" at 38% usage. The steady-average warning
  still shows; only the over-extrapolation is tamed.
- **Days instead of huge hours.** The weekly reset countdown and the "to limit"
  estimate switch to days past 24h (e.g. "5d 1h", "~2d 21h") instead of "121h".
- **Sleep-safe.** The estimate decays by real elapsed time, so it no longer
  shows a stale-high projection right after the laptop wakes.

## Performance

- Cost aggregation computes each record's cost once (cached) instead of
  re-deriving it ~6× per sync — less CPU for users with very large `~/.claude`
  history.

## Internal

- Added a Vitest unit-test suite for the projection / pace math, gating releases
  in CI alongside the existing Rust tests.

## Notes

- All costs are *estimates* from local JSONL × official pricing (subscribers are
  not billed per-token).

## Download

### Windows
- `Claude Widget_2.4.2_x64-setup.exe` — NSIS installer (recommended)
- v2.0.x and later: Settings → "Check for updates" for automatic delivery

### macOS (Apple Silicon)

- macOS 11 Big Sur or later (Apple Silicon)
- WebView2 Runtime (auto-installed by the Win10 installer)

## 변경 — 한도 투영

- **알아서 완화.** 버스트의 공격적인 "한도 도달 예상"이 동기화 사이에 시간이 지나며
  스스로 누그러집니다 — 계속 새로고침할 필요 없음.
- **주간 버스트 상한** = 주중 평균의 2배. 짧은 스파이크가 38%인데 "한도까지 7시간"으로
  뜨지 않습니다. 평균 기반 경고는 그대로, 과대 외삽만 완화.
- **큰 시간 대신 일(日).** 주간 초기화 카운트다운과 "한도까지"가 24시간 초과 시 일
  단위로(예 "5일 1시간", "~2일 21시간") 표시됩니다 — "121시간" 아님.
- **절전 안전.** 실제 경과 시간만큼 감쇠해, 노트북이 깨어난 직후 stale 투영이 안 뜸.

  큰 `~/.claude` 히스토리 사용자의 CPU 부담 감소.
