# v2.4.3 — Compact normal view & floating error banner

A small UI-polish release: the normal view is tidier and a bit more compact, and
the rate-limit/error banner now floats over the content instead of pushing it down.

## Changed

- **Floating error banner.** A rate-limit/error banner now overlays the top of
  the content (like the toast) instead of pushing the donut and metrics down —
  the header stays visible.
- **Tidier, more compact normal view.** The session and weekly sections are
  tightened and both "resets in …" captions sit with matching top/bottom
  spacing, so the normal widget is a touch shorter and more even.

## Internal

- The in-app guide mirrors the real widget size from a single source, so it
  can't drift from the actual layout.

## Notes

- All costs are *estimates* from local JSONL × official pricing (subscribers are
  not billed per-token).

## Download

### Windows
- `Claude Widget_2.4.3_x64-setup.exe` — NSIS installer (recommended)
- v2.0.x and later: Settings → "Check for updates" for automatic delivery

### macOS (Apple Silicon)
- `Claude Widget_2.4.3_aarch64.dmg` — DMG installer
- First-launch Gatekeeper bypass: right-click → Open, or
  `xattr -d com.apple.quarantine "/Applications/Claude Widget.app"`

## System requirements

- Windows 10/11 (Win11 recommended — Mica vibrancy)
- macOS 11 Big Sur or later (Apple Silicon)
- WebView2 Runtime (auto-installed by the Win10 installer)
